USE CASE:
Using Red Hat Ansible Automation Platform for edge computing 

Standardize configuration and deployment across your entire IT landscape—all the way to the edge—with a single, consistent automation platform.

Read the IDC Spotlight

You’re already at the edge. Now, automate it.

Red Hat Ansible Automation Platform uses containerization to distribute and run automation across environments. This design makes running automation in environments with limited resources possible—letting you automate better at the edge.

Scale rapidly

Automation mesh rapidly scales capacity for local and remote automation workloads to meet automation volume demands.

Reduce risk

Native features like hop nodes introduce fault tolerance and redundancy capabilities to reduce the impact of external dependencies.

Execute reliably

Automated health checks determine the optimal route and node to run automation jobs based on key metrics before sending staff into the field.

Scale with improved security

Execute the automation workflows closer to the edge endpoints through an encrypted and distributed architecture, while keeping a centralized control layer with enterprise-grade capabilities.

Localized execution

Increase resiliency to high latency and connection disruptions by running automation closer to the endpoints.

Enterprise-wide automation

Manage your infrastructure across vendors, geographies, and computing environments where there is little or no IT staff.

Enterprise automation. But really far away.

Red Hat® Ansible® Automation Platform reaches and automates distributed workloads just as it automates database, network, security, applications, AI/ML and server-side workloads: using blueprints of automation tasks within certified collections of playbooks.

Automation mesh is a flexible and scalable architecture that eases the distribution of work across a large and dispersed collection of workers by connecting remote execution nodes to a centralized automation controller.

Ansible Automation Platform allows you to automate: 

  • Distributed computing elements.
  • Inventory IoT device endpoints connected at the far edge.
  • Network edge devices.
  • Application Lifecycle on Windows and Linux endpoints.
  • Actions in response to user input at the infrastructure edge.

Automation at the edge of whatever you do

Retail and restaurants

Automate endpoints to simplify infrastructure operations, reinforce security policies, and standardize device configuration across branches. Plus, reduce the time to deploy IT resources when expanding or relocating.

Transportation

Automate the lifecycle of edge devices with updates delivered to field devices as needed with minimal disruption to users and operations teams.

Financial services

Automate software patches, configuration compliance and security policies to reinforce network security across devices and remote locations.

Industrial

Automate configurations of virtualization, IoT devices, networks, and applications in industrial and manufacturing facilities.

Healthcare

Configure and manage healthcare devices, data collection, and networks across your entire healthcare infrastructure.

Energy

Simplify applications updates and solve dependencies including certificates and licenses management across your HMI devices.
